# Anime Assistant 开发日志 day19

日期：2026-07-11

## 今日目标

对 Anime Assistant 项目进行完整架构审查，并逐项解决状态一致性、旧数据兼容、线程安全、配置管理、隐私保护、AI 输出校验和响应性能问题。

## 完成内容

### 1. 项目全貌审查

完整检查了项目目录、README、配置文件、28 个原始 Python 模块、运行数据结构、Live2D 诊断脚本和虚拟环境依赖。

梳理了控制台、GUI、ConversationOrchestrator、InitiativeEngine、记忆检索、情绪关系状态和 Live2D 之间的调用关系。

### 2. 修复共享聊天历史分叉

原先 Orchestrator 和 InitiativeEngine 在第一次保存历史后会持有不同列表，主动消息可能使用旧历史覆盖新对话。

现已将历史清洗和截断改为原地更新，确保所有组件始终共享同一个历史对象。

### 3. 增加旧事件数据迁移

旧事件缺少 `id`、`created_at`、`embedding` 和 `notified`，可能导致主动消息生成后因 `KeyError` 丢失。

新增统一事件迁移层：

- 自动补齐唯一 ID。
- 处理重复 ID 和非法数据。
- 保留未知发生时间，不伪造成新事件。
- 缺少 ID 时主动消息仍能正常记录。

### 4. 收紧线程与关闭流程

正常回复现在使用聊天历史和上下文快照，避免后台线程修改正在生成回复的数据。

主动消息改为：

1. 锁内计算触发条件。
2. 锁外调用 AI。
3. 锁内复核用户是否已经互动。
4. 只提交仍然有效的主动消息。

GUI 关闭时不再销毁仍在运行的 QThread，而是等待当前聊天线程自然结束后再释放资源和 Live2D。

### 5. 统一配置与项目路径

新增统一项目路径模块，使配置、数据、日志和 persona 不再依赖当前工作目录。

新增 AI 客户端工厂，集中管理 OpenAI 兼容服务地址。

删除生产代码中的：

- `C:\mio\...` 默认模型路径。
- 写死的用户名称 `Lucas`。
- 分散的 DeepSeek 地址。
- 控制台和 GUI 中重复的主动聊天参数。

Live2D 路径无效时会自动退化为纯聊天界面。针对现有用户配置，已显式恢复本地模型路径：

`C:\mio\mio\MIO.model3.json`

### 7. 收紧 Git 隐私边界

Git 已停止跟踪：

- 用户聊天历史。
- 用户画像与关系状态。
- 事件和长期摘要。
- `.bak` 备份。
- `venv312` 虚拟环境。
- `.idea` 本地 IDE 配置。

本地文件全部保留。版本库只继续跟踪 `data/persona.json` 和 `data/.gitkeep`。

需要注意：停止跟踪不能自动清除旧 Git 历史。如果仓库以前已经公开，后续应单独评估是否使用 `git filter-repo` 清理历史中的用户数据。

### 8. 统一数据模型与 AI JSON 校验

新增 `data_models.py`，统一校验：

- 应用配置。
- 用户资料。
- 情绪与关系状态。
- 对话消息。
- 事件和长期摘要。
- 主动消息状态。
- 意图识别结果。
- 资料提取结果。
- 事件提取结果。

AI 即使返回合法但类型错误、枚举错误或数值越界的 JSON，也不会再污染业务状态。

同时修复了 `profile_manager.py` 中 `"啥"` 后缺少逗号的问题。

### 9. 优化回复性能

针对首字延迟和回复后卡顿完成五项专项优化：

- 增加 `[PERF]` 分阶段性能日志。
- 在后台预热中文语义模型。
- 模型未就绪时使用轻量词面检索。
- 无 embedding 的事件继续使用词面检索，不会失忆。
- 主回复完成后立即恢复输入。
- 事件提取、情绪和关系更新进入顺序后台队列。
- 长期摘要改为每累计 10 条溢出消息批量执行。
- 待摘要消息持久化到 `data/pending_summary.json`。
- 事件 JSON 增加线程安全内存缓存，文件未变化时不再重复解析约 445 KB 数据。

性能日志现在可以区分：

- 输入准备时间。
- 事件文件读取和缓存时间。
- 语义或词面检索时间。
- Prompt 构建时间。
- API 连接时间。
- 首字时间 TTFT。
- 完整流式回复时间。
- 事件提取时间。
- 长期摘要时间。
