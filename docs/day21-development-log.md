# Day21 开发日志

日期：2026-07-12

## 今日目标

为 Anime Assistant 接入可用的日语语音输出，让 MIO Live2D 在等待和说话时表现得更自然；同时解决 DeepSeek V4 空回复、错误话术污染历史和角色口吻偏离秋山澪的问题，并统一 Python 3.14 开发环境。

## 完成内容

### 1. AivisSpeech 日语语音接入

- 新增独立 `tts_service.py`，通过本地 AivisSpeech Engine 生成日语语音。
- 中文回复先在后台转换为自然日语，再交给 AivisSpeech 合成。
- 根据平静、开心、害羞、低落和疲惫选择不同的语音 Style ID。
- 长回复会按句子安全拆分，所有片段成功后合并为一条完整 WAV，再统一播放。
- 任一片段失败或超时时取消整条语音，但保留文字回复，不阻塞正常聊天。
- 增加语速、音量、请求超时、单段字符数和心情音色映射配置。

### 2. TTS 播放稳定性与真实音频嘴型

- 日语翻译、语音合成和音频准备全部放入后台线程，避免冻结 PySide6 主界面。
- 修复 QtMultimedia 音频源切换时的重入问题，避免播放几句话后出现 `AppHangTransient`。
- 完整回复只创建一次播放源，减少分段切换造成的卡顿和后半段降级。
- 从 WAV 短时响度生成嘴型包络，让嘴巴跟随真实语音强弱开合。
- 语音不可用时自动回退为纯文字，不影响 Live2D 和输入框恢复。

### 3. Live2D 等待状态与可调待机动作

- 语音生成期间进入独立等待状态，加入平滑视线游移、眉毛微动、头部摆动和身体跟随。
- 利用模型物理系统，通过头部角度带动头发，而不是直接写入不存在的头发参数。
- 加强待机头部左右摆动与头发响应，同时保留启动渐入，降低刚启动时抖动概率。
- 在“调表情”窗口新增以下实时滑块：
  - 待机摆头与头发强度；
  - 待机视线游移强度；
  - 待机动作速度。
- 三项参数均可独立预览和保存，不会重置当前动作相位。

### 4. Python 3.14 项目环境统一

- 创建项目专用 `.venv`，固定使用 Python 3.14.2。
- 将 `live2d-py` 从不支持 Python 3.14 wheel 的 `0.6.1.1` 更新为 `0.7.0.4`。
- 补齐 PySide6、OpenAI、PyTorch、Sentence Transformers 和 Live2D 运行依赖。
- 新增 VS Code 项目设置，默认选择 `.venv`、自动激活终端并识别 unittest。
- 以后可以直接使用 `python main_gui.py` 和 `python -m unittest ...`，无需每次指定 `py -3.14`。

### 5. DeepSeek V4 空回复修复

- 定位到 `deepseek-v4-pro` 默认开启思考模式，短回复的 `120` token 预算可能被思考内容消耗，导致最终 `content` 为空。
- 日常主对话显式关闭思考模式，后台事件和长期记忆任务保持原行为。
- 将日常回复预算调整为 `72` tokens，使回复更简短并降低延迟。
- 空流第一次出现时自动重试；第二次仍为空时显示角色内兜底，避免界面静默结束。
- 日志新增 `reasoning_chars`、正文状态和 `finish_reason`，可以区分预算耗尽、服务端空流和真实网络异常。

### 6. 兜底话术历史污染防护

- 发现过去的网络兜底句被当作正常助手回复保存，模型在网络恢复后继续模仿相同错误话术。
- 将系统兜底话术集中到 `ai/fallbacks.py` 管理。
- 兜底回复现在只显示在界面，不写入短期历史，也不触发事件提取、情绪更新和长期摘要。
- 加载历史时自动过滤以前保存的精确兜底句。

### 7. 秋山澪人设与自然女高中生口吻校准

- 重新整理人物身份、性格和说话方式：内向怕羞、认真可靠、体贴但有原则，也会无奈、嘴硬、吐槽和温柔地反对。
- 明确角色首先是有自己生活和判断的女高中生，而不是永远附和的客服。
- 日常回复通常控制在 1～2 句、约 12～55 个汉字。
- 禁止括号动作、小说旁白、景色铺陈和凭空编造校园或演出经历。
- 只有正在讨论音乐时才提贝斯、琴弦、低音或轻音部，避免所有情绪都使用音乐比喻。
- 主回复只参考最近 8 条消息；旧历史只提供事实连续性，不作为文风示例。
- 开场白和主动消息同步采用简短口语，并增加最终动作清理和长度保护。
- 修复“你喜欢什么样的人”被资料查询路由错误回答为“你喜欢：暂无”的问题。

### 8. 旧风格历史安全迁移

- 审查发现旧助手回复平均约 138 字，大量包含贝斯比喻和小说式细节，已经形成明显风格污染。
- 在迁移前备份原有 47 条短期消息。
- 重置短期对话历史，为新口吻建立干净上下文。
- 用户资料、情绪、关系数值、事件记忆和长期摘要全部保留。
- 后续启动通过迁移版本标记避免重复重置。

## 关键问题与结论

### AivisSpeech 只播放前半段

根因是长回复被拆成多个合成请求，其中后续片段可能超时。现在改为所有片段生成成功后先合并完整 WAV，再开始播放；失败时整条语音取消，不再出现只播放前半段的状态。

### Live2D 播放后界面未响应

根因是 QtMultimedia 在状态变化回调中立即切换和释放音频源，引发重入。现在将音频源切换延迟到下一轮事件循环，并在释放缓冲区前主动解除播放器源。

### 连续出现“网络不稳定”回复

最初确实存在空 `content`，但关闭思考后日志已经显示正常正文。后续重复句来自历史模仿而非网络失败，因此增加了兜底历史隔离和旧记录过滤。

### 说话方式不像秋山澪

系统提示已经要求简短，但旧历史中 21 条助手回复平均约 138 字，其中 15 条包含贝斯、琴弦或低音表达。具体旧回复比抽象规则更容易被模型模仿，因此最终采用“重写人设规则 + 限制短期上下文 + 备份并重置污染历史”的组合方案。
