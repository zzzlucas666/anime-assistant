## 今日目标

在 Day27 完成可信记忆治理后，重新整理角色提示系统：把固定人格、价值观、关系行为、动态上下文和输出约束分开管理，减少规则堆叠与入口差异。同时根据真实测试修复“我很难受”未触发 Mio 担心反应，以及担心语音情绪混杂和偶发失真的问题。

## 完成内容

### 1. 建立五层 Prompt 架构

- 将原本集中在 `ai/chat.py` 中的大段规则拆分为五个职责稳定的层次：
  - `Identity`：固定身份、性格倾向和自然表达方式；
  - `Values`：真诚、陪伴、事实边界和温和纠错等选择原则；
  - `Behavior`：由当前关系与情绪推导出的温暖度、开放度、主动程度、玩笑程度和认真程度；
  - `Dynamic Context`：当前情绪、关系阶段、用户资料和分级可信记忆；
  - `Output Rules`：长度、口语风格、事实边界和机器可读情绪标签。
- 新增 `anime_assistant/ai/prompts/` 功能包，每一层均可单独测试和维护。
- 使用统一 composer 按固定顺序组合五层内容，避免以后新增规则时重新形成单个超大 Prompt。

### 2. 统一普通聊天、启动问候和主动聊天

- 三种入口现在共用同一个五层 Prompt 构造器。
- 启动问候与主动聊天只改变场景上下文和输出约束，不再维护独立的人格副本。
- 主动聊天不会假装用户刚刚说过一句话，启动问候也不会误用普通对话中的即时情绪控制标签。
- Identity 与 Values 在所有场景保持固定，动态变化只发生在 Behavior 与 Context 层。

### 3. 将关系数值转换为语义行为

- 不再把原始 affection、trust 和 familiarity 分数直接交给模型。
- Python 先将关系状态转换为“关系阶段、温暖程度、表达开放度、主动程度”等语义行为。
- 模型只负责自然表达，不再自行猜测某个数值应该对应多亲密的称呼和行为。
- 关系策略继续复用 Day27 统一后的权威实现，避免提示层产生第二套关系逻辑。

### 4. 为动态记忆增加可信等级

- 用户明确表达并有证据的事实作为高可信记忆进入 Prompt。
- 系统观察和旧版迁移记录作为中可信背景，并明确要求不要补充细节。
- AI 推测、候选、过期、撤回和被替代记录继续完全排除。
- 长期摘要被标记为模糊背景，不允许覆盖本轮用户原话或高可信档案。

### 5. 强化情绪控制输出契约

- 普通聊天继续使用用户不可见的 `<mio:...>` 控制标签，让 AI 在本地候选基础上校准混合情绪。
- 输出规则明确要求自然回复结束后不可省略控制标签。
- 用户难过、焦虑或疲惫时，继续区分“用户的情绪”和“Mio 的反应”：用户可以是 `sad`，Mio 应表现为 `worried`，而不是跟着变成 `sad`。
- 本地情绪规则仍作为安全底线，即使模型偶尔未返回标签，明确表达也能触发表情与语音。

### 6. 修复“难受”未触发担心

- 补充“我难受、我很难受、今天真的很难受、感觉很难受”等表达。
- “我今天真的很难受，但暂时不想说原因”现在会稳定得到：
  - 用户情绪：`sad`；
  - Mio 持续心情：`neutral`；
  - Mio 短暂反应：`worried`；
  - 语音风格：`concerned`。
- 同时增加否定识别，“我已经不难受了”不会错误触发担心。
- 将这一真实问题加入固定对话回归集，防止后续重构再次遗漏。

### 7. 修复担心语气异常与偶发失真

- 定位到原 `concerned` 与 `reassuring` 共用一条混合参考音频，其中同时包含安慰、害怕和厌恶语气，导致 GPT-SoVITS 模仿的韵律不稳定。
- 将两种支持型语气拆开：
  - `concerned` 使用更轻、犹豫但干净的参考；
  - `reassuring` 使用更温和自然的参考。
- GPT-SoVITS 输出写入 WAV 前增加峰值余量保护，以整体缩放替代硬削波，降低高峰处的沙裂和失真。
- 本地 `settings.json` 已同步新参考，但继续保持 Git 忽略，不提交 API Key 和机器专属路径。

### 8. 文档与回归测试

- README 增加五层 Prompt 架构和可信动态上下文说明。
- 架构文档补充五层构造顺序、各层边界和不同对话模式的共享关系。
- 新增 Prompt 架构测试，验证层次顺序、固定层稳定性、动态行为变化、可信记忆分级和 Prompt 大小边界。
- 扩展情绪、TTS、对话风格和固定对话回归测试，覆盖“难受 / 不难受”、担心与安慰参考分离、控制标签不可省略以及音频峰值保护。

## 关键结论

### 为什么要分成五层

固定人格和动态情绪的生命周期完全不同。把它们堆在一个 Prompt 中，会让短暂状态逐渐污染人格，也会让相同规则在问候、主动聊天和普通聊天之间漂移。五层结构让稳定内容保持稳定，让动态内容每轮按状态重新生成，同时保留一个统一输出契约。

### 为什么不能只依赖 AI 判断情绪

模型返回的隐藏标签可以处理夸奖、害羞与开心等模糊语义，但不能成为唯一入口。网络异常、输出截断或模型偶尔省略标签时，本地明确规则必须继续识别“难受、紧张、疲惫”等高价值信号，保证表情和语音不会完全失效。

### 为什么担心和安慰要使用不同参考

`concerned` 表达的是认真关注和轻微不安，`reassuring` 表达的是稳定陪伴与安慰。两者共用一条同时包含恐惧和厌恶的参考，会让生成语气摇摆并放大音色瑕疵。拆分后即使仍使用同一个模型，韵律目标也更一致。
