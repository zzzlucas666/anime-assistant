# Day22 开发日志

日期：2026-07-14

## 今日目标

使用已获得授权的秋山澪原生语音制作第一版专属日语 TTS 模型，在有限数据量和 8GB 显存条件下完成高质量数据整理、Style-Bert-VITS2 训练与对比试听；随后将 2000 步模型安全接入 Anime Assistant，同时保留 AivisSpeech 降级能力，并确保 Python 3.14 的 GUI / Live2D 环境不受训练依赖影响。

## 完成内容

### 1. Mio 原始语音整理与高精度转录

- 确认原始素材位于本机 `mio_voice` 目录，总时长约 11 分 58 秒。
- 素材已经人工确认不包含其他角色声音、背景音乐或明显环境杂音。
- 使用 Whisper Large V3 完成日语语音转录，优先保证准确率而不是下载和处理速度。
- 保留完整转录结果，后续可以继续从未入选片段中补充第二版训练数据。
- 所有原始音频、转录中间产物和训练数据均保存在本地 `data/`，不进入 Git 仓库。

### 2. 第一版高质量训练集构建

- 从转录片段中进行保守筛选，优先保留发音完整、语义明确、停顿自然的片段。
- 第一版训练集最终包含 48 条音频，总时长约 251.1 秒，即约 4 分 11 秒。
- 单条音频时长控制在约 2.16～9.98 秒，符合 Style-Bert-VITS2 的训练习惯。
- 划分为 46 条训练数据和 2 条验证数据。
- 生成并检查以下训练文件：
  - `esd.list` 与清洗后的文本；
  - `train.list` 和 `val.list`；
  - 48 份 BERT 特征；
  - 48 份语音风格特征；
  - 数据选择报告和排除片段报告。
- 151 个不够稳定的候选片段没有进入第一版，以降低错字、漏字、截断和噪声对音色的污染。

- 将本地路径、训练数据路径和模型资产路径集中到独立配置中，不污染主程序依赖清单。

### 4. 1000 步第一版训练与试听

- 使用 JP-Extra 底模、批大小 1 开始训练。
- 首次尝试在反向传播阶段出现显存不足，定位到 8GB 显存同时承担生成器、判别器和 WavLM 判别器的压力。
- 启用冻结解码器策略，并使用可扩展 CUDA 显存分配，显存占用稳定在约 6.3～6.8GB。
- 训练期间持续监控显存、温度、生成器损失、Mel 损失和 KL 损失。
- 在第 1000 步保存生成器、判别器和 WavLM 判别器检查点，并导出推理用 `.safetensors`。
- 生成中性、害羞和音乐话题三条固定测试语音，确认模型可以稳定输出 44.1kHz 单声道 WAV。

### 5. 续训到 2000 步并保留对比版本

- 保存 `G_2000.pth`、`D_2000.pth`、`WD_2000.pth` 和推理模型 `mio_pilot_v1_e43_s2000.safetensors`。
- 使用与 1000 步完全相同的文本和推理参数生成对比试听，避免测试文本差异影响主观判断。
- 1000 步与 2000 步推理模型均保留，便于随时回退和继续比较。
- 三条 2000 步试听音频通过时长、采样率、削波比例和直流偏移检查。

## 关键问题与结论

### 第一版只有约 4 分钟数据，是否足够

这批数据足以制作可试听、可接入项目的第一版模型，但不足以覆盖秋山澪完整的语气变化。当前策略是先以 2000 步作为稳定基线，在实际聊天中观察音色、吐字和韵律，再将高质量数据扩充到 15～30 分钟。相比直接盲目增加训练步数，补充准确、多样的训练数据更有价值。

### 为什么不直接在主程序环境导入 Style-Bert-VITS2

主程序依赖 Python 3.14、PySide6 和 `live2d-py`，训练与推理栈使用 Python 3.10、特定版本 PyTorch 和 Transformers。强行合并环境会增加依赖冲突和安装失败风险。常驻子进程桥接让两套环境保持隔离，同时只在后台传递文本和 WAV。

### 为什么训练时冻结解码器

未冻结时首次反向传播超过 RTX 4060 Laptop 的可用显存。冻结 JP-Extra 解码器后，训练重点放在说话人和文本相关层，同时保留成熟底模的解码质量，使 8GB 显存上的第一版训练稳定完成。

### 为什么不把模型上传到 GitHub

推理模型约 251MB，训练检查点更大，训练数据还包含本地授权音频。这些文件全部位于 `.gitignore` 覆盖的 `data/` 目录。GitHub 只保存可复用的桥接代码、配置模板、测试和文档，实际权重与数据继续由本机管理。

### 当前情绪为什么还没有不同声线

第一版数据没有按情绪建立独立 Style，因此目前所有心情共用 `Neutral` 风格。Live2D 表情仍会响应心情；语音多风格需要在第二版数据中分别整理平静、害羞、开心、认真等类别后再训练。

- 2000 步模型输出为有效的 44.1kHz、16-bit、单声道 WAV。
- Python 3.14 → Python 3.10 → Style-Bert-VITS2 跨环境端到端合成成功。
- 语音队列、完整 WAV 合并和真实音量嘴型包络回调通过实测。
