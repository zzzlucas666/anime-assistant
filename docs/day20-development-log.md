# Day20 开发日志

日期：2026-07-12

## 今日目标

改善 MIO Live2D 角色的情绪表现、待机生动度和说话嘴型，并为不同模型参数提供可视化调试能力。

## 完成内容

### 1. 程序化角色表现

- 将持久情绪、说话状态和短暂强调拆分为不同表现层。
- 加入自然眨眼、视线变化、呼吸和轻微身体动作。
- 为问号、感叹号和波浪号增加短暂的歪头、抬眉或俏皮表现。
- 情绪参数改为平滑过渡，避免突然换脸或参数残留。

### 2. Live2D 启动与头部防抖

- 修复渲染周期和参数刷新周期不一致造成的隔帧姿态跳变。
- 每次 OpenGL 绘制都重新写入角色参数，覆盖启动、缩放和额外重绘帧。
- 启动后的前两秒逐渐引入头部与身体微动，让模型物理系统先完成稳定。
- 降低头部微动幅度和频率，使待机动作更加柔和。

### 3. 情绪表情校准

- 根据 MIO 模型的实际截图重新校准开心、低落、害羞和疲惫参数。
- 取消容易形成怒眉的眉毛角度组合，降低眉毛高度和形变幅度。
- 表情强度改为相对中性值放大，确保低落和疲惫时眼睛会正确闭合。
- 调整嘴型、眼睛、脸颊和头部姿态的组合，减少表情之间的冲突。

### 4. 表情参数调试窗口

- 在主界面状态栏增加“调表情”入口。
- 支持平静、开心、低落、害羞和疲惫五种情绪预设。
- 提供眼睛、视线、眉毛、嘴唇、脸颊、头部和身体共 19 个实时滑块。
- 支持恢复推荐值、全部归中和保存当前情绪。
- 保存结果写入本地 `config/settings.json`，关闭窗口后自动退出手动预览。
- 更新 `.gitignore`，防止本地配置及其备份中的 API 密钥被提交。

### 5. 嘴型节奏与持续播放

- 用文本节奏嘴型替换固定正弦波开合。
- 每个字符拆分为张开和回落两个阶段，平均速度约为每秒 5～7 个汉字。
- 为逗号、句号、问号和感叹号加入不同长度的停顿。
- 文本流结束后继续播放剩余嘴型队列，避免 AI 返回较快时只张几下嘴。
- 新增静态嘴唇开合参数，并与说话嘴型平滑叠加。

## 模型检查结论

- MIO 模型没有独立的 `exp3.json` 和 `motion3.json`，当前主要通过参数驱动表现。
- `ParamMouthForm` 和 `ParamCheek` 的程序写入及模型读回均正常。
- 嘴型和额外脸红变化不明显，主要受 `.moc3` 内部关键形状和透明度绑定幅度限制。
- 基础脸颊粉红线已经绘制在贴图中，无法仅靠提高 `ParamCheek` 数值显著增强。

## 验证结果

- Live2D 控制器专项测试全部通过。
- 项目完整测试共 45 项，全部通过。
- PySide6、OpenGL 和 live2d-py Python 3.14 运行环境检查通过。
- 参数调试窗口的预览、保存、恢复和关闭清理流程通过验证。
